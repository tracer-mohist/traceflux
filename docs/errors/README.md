## How to Use This Log

### When Something Goes Wrong

1. **Don't panic** — Errors are normal
2. **Document immediately** — Memory fades
3. **Use template** — Consistent format
4. **Share with team** — Collective learning

### Template

```markdown
# Error Log: [Brief Description]

**Date**: YYYY-MM-DD  
**Error Type**: [Process/Technical/Configuration/etc]  
**Severity**: [Low/Medium/High/Critical]  
**Resolution**: [How it was fixed]

## What Happened

[Describe the situation]

## Why It Failed

[Root cause analysis]

## Correct Approaches

[How to do it right]

## Lessons Learned

[Key takeaways]

## Commands Reference

[Useful commands for debugging]

## Timeline

[Chronological events]
```
